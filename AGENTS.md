# Git Commit Convention

提交信息遵循 [Conventional Commits](https://www.conventionalcommits.org/) 规范：

## 格式

```
<type>: <简短描述>
```

## Type

| Type | 用途 |
|------|------|
| `feat` | 新功能 |
| `fix` | Bug 修复 |
| `refactor` | 重构（不新增功能也不修 bug） |
| `perf` | 性能优化 |
| `test` | 添加或修改测试 |
| `docs` | 文档 |
| `chore` | 构建、CI、依赖等杂项 |
| `style` | 代码格式（不影响逻辑） |
| `ci` | CI 配置变更 |
